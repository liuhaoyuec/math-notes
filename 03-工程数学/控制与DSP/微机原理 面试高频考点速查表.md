# 微机原理 面试高频考点速查表

## 1. 冯·诺依曼结构 vs 哈佛结构
**核心定义**：
* **冯·诺依曼结构**：程序指令和数据存放在**同一**存储器中，共用一条总线。
* **哈佛结构**：程序指令和数据存放在**不同**的存储器中，拥有独立的两套总线，可以同时取指和取数。

**疑惑点突破**：
* **现在的电脑到底是什么结构？** 现代微处理器（如 x86、ARM 架构）通常采用**混合结构**（改进型哈佛结构）：在 CPU 内部的 L1 Cache（一级缓存）层面是哈佛结构（分为指令 Cache 和数据 Cache），而在外部主存层面则是冯·诺依曼结构。

## 2. 核心存储器：SRAM 与 DRAM 的区别
| 对比维度 | SRAM (静态随机存取存储器) | DRAM (动态随机存取存储器) |
| :--- | :--- | :--- |
| **基本存储单元** | 触发器 (Flip-flop) | 电容 (Capacitor) |
| **速度** | 极快 | 较慢 |
| **集成度与成本** | 晶体管多，集成度低，成本高 | 晶体管少（单管），集成度高，成本低 |
| **是否需要刷新** | **不需要** | **必须定期刷新** |
| **主要应用场景** | CPU 内部的高速缓存 (Cache) | 计算机的主存 (内存条) |

**疑惑点突破**：
* **为什么 DRAM 必须刷新？** DRAM 利用电容存储电荷来表示 0 和 1。由于电容存在漏电现象，时间一长电荷会流失导致数据丢失，所以必须外加刷新电路，定期给电容补充电荷。

## 3. CPU 与外设的三种数据传送方式 (极高频考点)
**核心定义**：
1.  **程序查询方式 (Polling)**：CPU 不断循环测试外设状态。条件满足才传送数据。
2.  **中断方式 (Interrupt)**：外设准备好数据后，主动向 CPU 发请求；CPU 暂停当前任务，转去处理数据，处理完再返回。
3.  **DMA 方式 (直接存储器存取)**：数据传送完全由硬件 (DMA 控制器) 接管，**不需要 CPU 参与**，数据直接在内存和外设之间高速传输。

**疑惑点突破**：
* **中断和 DMA 到底有什么本质区别？** * 中断依然需要 **CPU 介入**（执行中断服务程序，通过寄存器周转数据），适合中低速设备。
    * DMA 是为了高速设备（如网卡、硬盘）设计的。DMA 发生时，**CPU 会交出总线控制权**，完全不参与数据搬运，速度仅受限于内存和总线的物理带宽。

## 4. 中断机制与中断向量表
**核心步骤**：中断请求 $\rightarrow$ 中断响应 $\rightarrow$ 保护现场 $\rightarrow$ 执行中断服务程序 $\rightarrow$ 恢复现场 $\rightarrow$ 中断返回。

**疑惑点突破**：
* **什么是“中断向量”？中断向量表里存的是什么？** * 这是一个极易混淆的概念。**中断向量表里存的绝对不是中断处理程序本身！** * 表里存的是各个中断服务程序的**入口物理地址**（即中断向量）。CPU 拿到中断号后，去表里查这个号对应的地址，然后把程序的指针（IP/CS 或 PC）跳到那个地址去执行代码。

## 5. 三总线结构 (System Bus)
**核心定义**：微机系统内部各部件之间传递信息的公共通道，分为三类：
* **地址总线 (AB, Address Bus)**：单向，决定了 CPU 能寻址的最大内存空间（例如 32位地址线对应 4GB 内存）。
* **数据总线 (DB, Data Bus)**：双向，决定了 CPU 一次能传送的数据位数。
* **控制总线 (CB, Control Bus)**：主要用来传送控制信号（如读/写命令）和时序信号。
