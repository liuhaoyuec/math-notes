## 一、 数学 $\Sigma$ 在内存中的真实形态

在没有高级框架的底层 C++ 内存架构中，数学上的求和符号 $\Sigma$ 以及自由指标，存在着绝对死板的映射关系：

1. **等号左边的标 (自由指标)：** 映射为代码**外层**的独立循环，负责划定结果数组的内存大小。
2. **$\Sigma$ 下方的标 (哑指标)：** 映射为代码**最内层**的累加循环，负责在内存中读取、相乘并覆盖。

---

## 二、 自由缩并的编译器：`torch.einsum`

为了避免手写深层嵌套循环，PyTorch 提供了 `einsum` (爱因斯坦求和约定)。它是将数学公式转化为 C++ 底层指令的最佳桥梁。

* **规则：** 直接输入索引字符串。出现在箭头 `->` 左边但在右边消失的字母，系统会自动为其在底层生成最内层的累加循环（即执行 $\Sigma$ 缩并）。

```python
import torch
A = torch.randn(10, 3, 4)
B = torch.randn(10, 4, 5)

# 对应数学公式：C_{bij} = Σ_k A_{bik} B_{bkj}
# 字母 k 在右边消失，触发 k 维度的累加缩并
C = torch.einsum('bik, bkj -> bij', A, B) 
```

## 三、 硬件极限特化版：`@` (Batched MatMul)

`@` 运算符（`torch.matmul`）是张量缩并在工程上的**特化版本**。为了迎合 GPU 张量核心 (Tensor Cores) 擅长做 2D 矩阵乘法的物理硬件特性，`@` 强制封印了高维自由度，采用了**“递归降维”**的设计哲学（对应数学路线 2）：

它强行将任意 $N$ 阶张量拆解为：`[批次维度集合, 倒数第2维, 倒数第1维]`。

**死板的执行逻辑：**
1. **只认尾巴：** 永远只有倒数第 1 维和倒数第 2 维有资格执行缩并（标准 2D 矩阵乘法）。
2. **平行宇宙 (广播)：** 尾巴之前的所有维度统统被看作外层的“循环次数 / 批次”。系统会自动通过复制（Broadcast）对齐左右两边的批次，然后在底层开启高并发的循环，执行几千次 2D 矩阵相乘。

```python
import torch

# A shape: [8, 1, 3, 4] -> 被看作 8 组 [3, 4] 矩阵
# B shape: [   10, 4, 5] -> 被看作 10 组 [4, 5] 矩阵
A = torch.randn(8, 1, 3, 4)
B = torch.randn(10, 4, 5)

# 发生广播：1 和 10 兼容，8 和空位兼容
# 结果会在前置维度 [8, 10] 这个大宇宙中，执行 80 次 [3, 4] @ [4, 5]
result = A @ B 

print(result.shape) # 输出: torch.Size([8, 10, 3, 5])
```
