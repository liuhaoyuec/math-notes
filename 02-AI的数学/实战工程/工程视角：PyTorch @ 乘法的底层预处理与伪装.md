## 一、 核心痛点：异阶张量的灾难
纯代数的 $\bmod$ 映射公式（见关联卡片）有一个致命限制：**要求 $A$ 和 $B$ 必须同阶（维度数量 $D$ 一致）。**
如果 $A$ 是 5 维，$B$ 是 2 维，数学公式直接崩溃。框架必须在计算前进行“偷天换日”。

---

## 二、 隐式预处理：引擎的“偷天换日”三步曲

当执行 `C = A @ B` 时，C++ 底层会瞬间执行以下“零拷贝”逻辑：

### Step 1: 尾巴核验 (最后两维)
* 提取 $A$ 的倒数第一维 和 $B$ 的倒数第二维。
* **铁律：** 必须绝对相等（或者其中一个是 1），否则直接报 `Shape Mismatch` 死刑。

### Step 2: 隐式左侧补齐 (Implicit Left-Padding)
* 找出 $A$ 和 $B$ 中维度阶数最高的一个（假设为 $N$ 阶）。
* 给较矮的张量，在**最左侧**疯狂垫上长度为 `1` 的伪维度，直到双方都是 $N$ 阶。
* *工程意义：低维张量（如二维权重矩阵）在毫秒间披上了高维的外衣，获得了和高维张量一起进入 $\bmod$ 宇宙大阵的资格。*

### Step 3: 批次融合与广播死锁 (Broadcasting)
* 经过 Step 2，双方已同阶。系统从右向左逐列对比批次维度：
    * `1` 碰 `N` $\rightarrow$ 维度融合成 `N`。（在底层触发 $\bmod 1$ 坐标锁死机制）。
    * `N` 碰 `N` $\rightarrow$ 维度融合成 `N`。
    * `M` 碰 `N` $\rightarrow$ **直接死锁报错**（物理宇宙无法对齐）。

---

## 三、 代码实战对账

```python
import torch

# A 是 4 阶 (批次为 10, 5)
A = torch.randn(10, 5, 3, 4)  
# B 只是 2 阶纯矩阵
B = torch.randn(4, 7)         

# 【底层伪装过程】：
# 1. B 被强行左侧补 1，篡改为 4 阶: [1, 1, 4, 7]
# 2. 批次对齐: [10, 5] 与 [1, 1] 融合为 [10, 5]
# 3. 触发 mod 公式: B 的坐标永远被锁在 [0, 0, k, j]

C = A @ B
print(C.shape) # 完美输出: torch.Size([10, 5, 3, 7])
```
