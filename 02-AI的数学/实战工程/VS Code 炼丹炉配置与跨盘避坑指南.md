## 💡 核心认知：什么是 `.ipynb` (Jupyter Notebook)？

在传统编程中，我们写 `.py` 文件，它像是一本**“印刷好的书”**。每次运行，电脑都必须从第一行重新读到最后一行。如果中间报错改了一个字，又要从头再来。

而 `.ipynb` (交互式笔记本) 是一种**“活的实验室”**，它是 AI 工程师绝对的标配：

1. **积木式执行 (Cell-based)：** 代码被分成了一个个灰色的“格子”(Cell)。你可以按 `Shift + Enter` 单独运行某一格。
2. **状态永驻内存 (Memory State)：** 这是最神仙的功能！假设你在第一格写了读取 10GB 数据的代码，运行完后，这 10GB 数据就永远留存在内存里了。你接下来在第二格、第三格修改模型参数、画图，都**不需要重新读取数据**！这能节约极其恐怖的时间。
3. **图文并茂：** 它可以像 Obsidian 一样把代码格切换成 Markdown 格式，让你在一份文档里同时拥有：数学公式 + 代码逻辑 + 实时输出的波形图。

## 🛠️ 第一阶段：VS Code 基础装配

1. **下载安装:** 从官网安装 Visual Studio Code。
2. **安装核心插件:** 在左侧边栏“扩展”中，搜索并安装由 Microsoft 官方提供的：
   * `Python` (提供代码高亮与解析)
   * `Jupyter` (提供 `.ipynb` 文件的界面支持)
3. **开辟战场:** 在电脑新建一个文件夹（如 `AI_Study`），用 VS Code 打开它，并在里面新建一个后缀名为 `.ipynb` 的文件（如 `01_test.ipynb`）。

## 💣 第二阶段：跨盘环境死锁大坑 & 终极破解法

**🔴 踩坑场景 (我们今晚遇到的惨痛教训)：**
当你的 Miniconda 和虚拟环境（如 `ai_base`）安装在了**非系统默认盘**（例如专门分出来的 Ubuntu E盘），而在 VS Code 里点击“选择内核”时，**右上角会出现无限循环的进度条，界面直接卡死或变为空白**。

**🔍 根本原因：**
1. VS Code 默认只在 C 盘疯狂寻址，跨越分区检索导致底层死锁。
2. 我们的 `ai_base` 沙盒里缺少了一座“跨海大桥”——名为 `ipykernel` 的通信包。

**⚡ 终极破解方案 (手动强推法)：**

**Step 1: 手动修桥 (Anaconda Prompt)**
打开 Anaconda Prompt 终端，进入沙盒环境，手动用清华源极速安装桥梁：
```bash
conda activate ai_base
pip install ipykernel -i [https://pypi.tuna.tsinghua.edu.cn/simple](https://pypi.tuna.tsinghua.edu.cn/simple)
```

**Step 2: 获取精确坐标**
在刚刚的黑框框里，继续输入：
```bash
where python
```
复制它吐出来的那条带有 `ai_base` 的具体路径（例如：`E:\ai\envs\ai_base\python.exe`）。

**Step 3: 暴力指路 (VS Code)**
1. 重启 VS Code，打开 `.ipynb` 文件。
2. 按 `Ctrl + Shift + P` 呼出万能指令台，输入并选择 `Python: 选择解释器 (Select Interpreter)`。
3. 选择 **“输入解释器路径...”**，把你刚才复制的精确路径直接粘贴进去，回车！
4. 此时右上角内核瞬间秒变 `ai_base (Python 3.10.x)`，死锁解除！

## 🚀 第三阶段：点火与操作快捷键

连接成功后，点击界面上的 `+ 代码` 即可新建代码格。

🔥 **高频快捷键 (必须肌肉记忆)：**
* **`Shift + Enter`**: 运行当前格子里的代码，并自动跳到下一个格子（最常用）。
* **`Ctrl + Enter`**: 只运行当前格子，不跳转。
* **按 `A` 键 (在格子左侧非输入状态下)**: 在当前格子**上方**新建一行代码框 (Above)。
* **按 `B` 键 (在格子左侧非输入状态下)**: 在当前格子**下方**新建一行代码框 (Below)。
* **按两次 `D` 键**: 删除当前代码格 (Delete)。
